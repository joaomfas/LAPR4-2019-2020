Introdução

Foi desenvolvido, no âmbito do 2º Semestre do 2º Ano da Licenciatura em Engenharia Informática (LEI), um Projeto Integrador. Este projeto foi desenvolvido tendo como objetivo a aplicação dos conhecimentos adquiridos em todas as Unidades Curriculares desse mesmo período. Este relatório irá focar-se nos objetivos relacionados com as competências adquiridas em Linguagens e Programação (LPROG), tais como:
— Definir uma representação XML para um domínio específico;
— Definir a sintaxe dessa linguagem através do respetivo XML Schema (XSD);
— Criar instâncias (documentos XML) e validá-las de acordo com o XSD;
— Fazer transformações ao documento XML (para HTML, JSON e TXT) usando XSLT e XPath.

O domínio definido pelo Projeto Integrador é relativo a um Sistema de Gestão de Ordens de Produção. O qual tem como objetivo tratar e armazenar a informação relativa ao decorrer de Ordens de Produção. A solução deve permitir ao cliente:
— Controlo de ordens de produção e respetiva execução;
— Gestão de tempo de atividade e paragem de máquinas;
— Gestão de consumos de matérias-primas;
— Registo de quantidades efetivamente produzidas.

Análise

Para dar resposta aos requisitos apresentados pelo Cliente, foi necessário começar por analisar o domínio com o qual se está a lidar. Sendo que, dessa análise pudemos destacar os seguintes pontos:
— Produtos;
— Matérias-primas;
— Máquinas;
— Linhas de produção;
— Ordens de produção;
— Execução das ordens de produção.
Tendo conhecimento das características destas entidades, foi possível definir quais os seus atributos e componentes para que a estrutura do ficheiro XML pudesse começar a ser pensada. Este processo foi feito concorrentemente com a elaboração do XSD, uma vez que é este o documento responsável pela validação da informação que consta no ficheiro XML.

Linguagem

Para a exportação de dados armazenados relativos ao domínio previamente referido foi utilizado um ficheiro XML. O XML (exTensible Markup Language) é uma linguagem de marcação que apresenta algumas parecenças com HTML. Esta linguagem tem como finalidade o armazenamento e transporte de informação e tem uma natureza auto-descritiva.[1]
O World Wide Web Consortium (W3C) começou, na década de 90, a trabalhar numa linguagem que permitisse combinar a flexibilidade da SGML com a simplicidade do HTML. Os príncipios do projeto eram criar uma linguagem que pudesse ser interpretada por software e que se integrasse com as restantes linguagens já existentes. Sendo que, as bases do XML seriam:[2]
— Separação do conteúdo da formatação;
— Simplicidade e legibilidade, tanto para humanos quanto para computadores;
— Possibilidade de criação de tags sem limitação;
— Criação de arquivos para validação de estrutura;
— Interligação de bancos de dados distintos;
— Concentração na estrutura da informação e não na sua aparência.

Para a validação dos ficheiros XML criados foi utilizada a linguagem XML Schema. Esta linguagem é baseada no formato XML e tem como objetivo a definição de regras de validação (esquemas) de documentos XML. Esta é uma alternativa ao DTD, cuja sintaxe não é baseada em XML, e foi a primeira linguagem (com este objetivo) a receber a recomentação por parte do W3C. [3]
Um ficheiro que descreve a estrutura de um documento XML recorrendo à linguagem XML Schema é denominado de XSD (XML Schema Definition).
Neste trabalho encontra-se este tipo de ficheiros em Relatorio.xsd ou ChaoDeFabrica.xsd. Sendo que, esses ficheiros apresentam ligações para outros ficheiros onde são estipuladas as regras para os tipos complexos e simples.

Transformações

Para transformar os documentos XML (relatório e informação do chão de fábrica) noutro tipo de ficheiros foram utilizadas duas linguagens distintas:[4]
- XSLT (eXtensible Stylesheet Language Transformations): Linguagem utilizada para transformar documentos XML;
- XPath (XML Path Language): Linguagem para navegar em ficheiros XML.
É importante observar que o documento XSL não altera o documento XML original, ou seja, não cria outro documento. Neste sentido, ele (o documento XSLT) atua como as folhas de estilos CSS: apenas determina como o browser deve apresentar o documento XML ao qual ele está associado ou anexado (de uma forma bem parecida com a usada para associar uma folha de estilos CSS a um documento (X)HTML). O novo documento contém as informações do documento XML original, na íntegra, como ele foi criado, porém, apresenta-as como o documento XSL determina. Por outras palavras, a conversão do documento XML através do XSL cria uma versão nova do documento num formato diferente para que este possa ser usado e interpretado em ambientes e contextos.

Conclusão

Uma vez que o XML foi elaborado com sucesso de acordo com a informação do domínio apresentado assim como com a sua validação recorrendo a ficheiros XSD e realizadas as transformações ao documento XML recorrendo a XSLT e XPath, consideramos que todos os objetivos foram concluídos com sucesso.
Consideramos ainda que a realização deste trabalho no âmbito de um Projeto Integrador permitiu a melhor consolidação dos conhecimentos adquiridos em LPROG através da aplicação dos conceitos e técnicas estudados aplicados a um "caso de vida real".
